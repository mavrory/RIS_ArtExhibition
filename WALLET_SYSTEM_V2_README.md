# Обновленная система оплаты - Только через кошелек

## Изменения от 2026-05-01

### Что изменилось

Система оплаты была упрощена и сделана более логичной:

#### ❌ Старая логика (убрано):
- Выбор способа оплаты при покупке произведения
- Прямая оплата картой/PayPal при покупке
- Фейковые транзакции для внешних платежей

#### ✅ Новая логика:

**1. Покупка произведений - ТОЛЬКО через кошелек**
- Пользователь может купить произведение только если у него достаточно средств на кошельке
- При покупке автоматически списываются средства с кошелька
- Создается транзакция типа PURCHASE
- Художник получает деньги на свой кошелек (транзакция ARTIST_EARNING)

**2. Пополнение кошелька - выбор способа оплаты**
- Пользователь идет в раздел "Wallet"
- Нажимает "Add Funds"
- Выбирает способ пополнения:
  - Credit Card
  - Debit Card
  - PayPal
  - Bank Transfer
  - Cryptocurrency
- Создается транзакция типа DEPOSIT

**3. Все транзакции имеют смысл:**
- DEPOSIT - пополнение кошелька (с карты, PayPal и т.д.)
- PURCHASE - покупка произведения (списание с кошелька)
- ARTIST_EARNING - заработок художника (зачисление на кошелек)
- REFUND - возврат средств (если будет реализовано)

## Обновленный процесс покупки

### Для покупателя:

1. **Пополнить кошелек:**
   ```
   Wallet → Add Funds → Выбрать способ оплаты → Ввести сумму → Подтвердить
   ```
   Результат: Баланс увеличился, создана транзакция DEPOSIT

2. **Купить произведение:**
   ```
   Artwork → Purchase Artwork → Pay Now (только если достаточно средств)
   ```
   Результат: 
   - Баланс покупателя уменьшился (транзакция PURCHASE)
   - Баланс художника увеличился (транзакция ARTIST_EARNING)
   - Произведение помечено как проданное

3. **Если недостаточно средств:**
   ```
   Artwork → Purchase Artwork → Видит предупреждение → Кнопка "Add Funds" → Wallet
   ```

### Для художника:

1. **Просмотр баланса:**
   - На Artist Dashboard видит:
     - Total Earned - общая сумма продаж
     - Current Balance - текущий баланс кошелька (кликабельно, ведет в Wallet)
     - Artworks - количество работ
     - Total Views - просмотры

2. **История заработков:**
   ```
   Wallet → Transaction History → Видит все ARTIST_EARNING транзакции
   ```

## Технические изменения

### Backend

**PaymentService.java:**
- Убрана логика обработки внешних способов оплаты
- Оставлена только оплата через WALLET
- Проверка баланса перед покупкой
- Понятное сообщение об ошибке при недостаточном балансе

```java
// Теперь только так:
if (currentBalance.compareTo(amount) < 0) {
    throw new BusinessException("Insufficient wallet balance. Current balance: $" + 
        currentBalance + ", Required: $" + amount + ". Please add funds to your wallet.");
}
```

### Frontend

**ArtworkDetailPage.tsx:**
- Убран компонент PaymentMethodSelector при покупке
- Показывается только текущий баланс
- Кнопка "Add Funds" если недостаточно средств
- Простой и понятный диалог оплаты

**WalletBalance.tsx:**
- Добавлен prop `refreshTrigger` для автообновления
- Баланс обновляется сразу после пополнения

**ArtistDashboardPage.tsx:**
- Добавлена карточка "Current Balance" с градиентом
- Кликабельна, ведет в Wallet
- Показывает актуальный баланс из API

**WalletPage.tsx:**
- Единственное место где можно выбрать способ оплаты
- Только для пополнения кошелька
- История всех транзакций с цветовой индикацией

## API изменения

**POST /api/payments/process**
- Параметр `paymentMethod` теперь игнорируется
- Всегда используется WALLET
- Возвращает ошибку если недостаточно средств

## Преимущества новой логики

1. **Понятность:** Один способ покупки - через кошелек
2. **Прозрачность:** Все транзакции имеют реальный смысл
3. **Безопасность:** Нет фейковых транзакций
4. **Простота:** Меньше кода, меньше багов
5. **Реалистичность:** Как в реальных маркетплейсах (Steam, App Store и т.д.)

## Тестирование

### Сценарий 1: Успешная покупка
1. Зарегистрируйтесь как пользователь
2. Перейдите в Wallet
3. Пополните на $100 через Credit Card
4. Проверьте, что баланс = $100
5. Найдите произведение за $50
6. Купите его
7. Проверьте, что баланс = $50
8. Проверьте историю транзакций (должна быть DEPOSIT и PURCHASE)

### Сценарий 2: Недостаточно средств
1. Баланс = $50
2. Попробуйте купить произведение за $100
3. Увидите предупреждение о недостаточном балансе
4. Кнопка "Pay Now" заблокирована
5. Нажмите "Add Funds"
6. Пополните кошелек
7. Вернитесь и купите

### Сценарий 3: Художник получает деньги
1. Войдите как художник
2. Проверьте баланс на Dashboard (должен быть $0)
3. Другой пользователь покупает вашу работу за $50
4. Обновите Dashboard
5. Current Balance = $50
6. Перейдите в Wallet
7. Увидите транзакцию ARTIST_EARNING на $50
