# Чеклист развертывания системы оплаты

## Перед запуском

### База данных
- [ ] Выполнить `add_balance_column.sql`
- [ ] Выполнить `add_wallet_system.sql`
- [ ] (Опционально) Выполнить `init_test_balances.sql`
- [ ] Проверить, что таблица `wallet_transactions` создана
- [ ] Проверить, что у пользователей есть колонка `balance`

### Backend
- [ ] Убедиться, что все новые Java файлы скомпилированы
- [ ] Проверить наличие пакета `com.digitalart.wallet`
- [ ] Запустить `mvn clean install`
- [ ] Запустить приложение `mvn spring-boot:run`
- [ ] Проверить логи на наличие ошибок
- [ ] Проверить доступность endpoints:
  - GET http://localhost:8080/api/wallet/balance
  - POST http://localhost:8080/api/wallet/deposit
  - GET http://localhost:8080/api/wallet/transactions

### Frontend
- [ ] Установить зависимости `npm install`
- [ ] Проверить компиляцию `npm run build`
- [ ] Запустить dev сервер `npm start`
- [ ] Проверить доступность страниц:
  - http://localhost:3000/wallet
  - http://localhost:3000/artworks/:id (диалог оплаты)

## Тестирование функционала

### Регистрация и вход
- [ ] Зарегистрировать нового пользователя
- [ ] Войти в систему
- [ ] Проверить, что кнопка "Wallet" видна в меню

### Кошелек
- [ ] Открыть страницу Wallet
- [ ] Проверить отображение баланса (должен быть $0.00)
- [ ] Нажать "Add Funds"
- [ ] Ввести сумму (например, $100)
- [ ] Выбрать способ оплаты (Credit Card)
- [ ] Подтвердить пополнение
- [ ] Проверить, что баланс обновился
- [ ] Проверить, что транзакция появилась в истории

### Покупка с недостаточным балансом
- [ ] Найти произведение дороже текущего баланса
- [ ] Нажать "Purchase Artwork"
- [ ] Выбрать способ оплаты "Wallet"
- [ ] Проверить, что появляется ошибка о недостаточном балансе
- [ ] Проверить, что кнопка "Pay Now" заблокирована

### Покупка с достаточным балансом
- [ ] Пополнить баланс на достаточную сумму
- [ ] Выбрать произведение
- [ ] Нажать "Purchase Artwork"
- [ ] Выбрать способ оплаты "Wallet"
- [ ] Проверить, что баланс отображается корректно
- [ ] Подтвердить оплату
- [ ] Проверить, что покупка прошла успешно
- [ ] Проверить, что баланс уменьшился
- [ ] Проверить транзакцию PURCHASE в истории

### Покупка другими способами
- [ ] Попробовать купить через Credit Card
- [ ] Попробовать купить через PayPal
- [ ] Проверить, что симуляция работает (задержка 2 сек)
- [ ] Проверить, что покупка проходит успешно

### Для художника
- [ ] Войти как художник (или создать произведение)
- [ ] Дождаться покупки произведения

## Проверка безопасности

- [ ] Попробовать открыть /wallet без авторизации (должен редирект на login)
- [ ] Попробовать пополнить отрицательную сумму (должна быть ошибка)
- [ ] Попробовать пополнить сумму > $10,000 (должна быть ошибка)
- [ ] Проверить, что транзакции показываются только текущему пользователю

## Известные ограничения

- Внешние способы оплаты (Credit Card, PayPal и т.д.) - это симуляция
- Нет реальной интеграции с платежными системами
- Нет системы возвратов
- Нет email уведомлений о транзакциях

## Если что-то не работает

### Backend не запускается
1. Проверьте, что PostgreSQL запущен
2. Проверьте настройки подключения в application.properties
3. Проверьте логи на наличие ошибок компиляции
4. Убедитесь, что все SQL скрипты выполнены

### Frontend не компилируется
1. Удалите node_modules и package-lock.json
2. Выполните `npm install` заново
3. Проверьте версию Node.js (должна быть >= 14)
4. Проверьте, что все импорты корректны

### Ошибка "Insufficient balance"
1. Проверьте баланс пользователя в БД
2. Убедитесь, что баланс >= цены произведения
3. Пополните кошелек через страницу Wallet

### Транзакции не отображаются
1. Проверьте, что таблица wallet_transactions создана
2. Проверьте, что API endpoint /api/wallet/transactions работает
3. Проверьте консоль браузера на наличие ошибок
